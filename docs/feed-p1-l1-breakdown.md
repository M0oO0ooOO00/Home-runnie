# FEED P1 + L1 작업 분해

> **목표**: 토스 커뮤니티형 무한 스크롤 피드(P1)를 비로그인 열람 가능(L1)한 형태로 출시한다.
>
> **컨셉**: 국내 야구 커뮤니티. 회원가입 진입 장벽 완화를 위해 L1(열람 익명)부터 도입, 4~6주 운영 후 지표 기반으로 L2(작성 익명) 도입 여부 결정.

---

## 0. 사전 합의된 결정사항

- **L1 = 열람 익명만**. 좋아요/댓글/팔로우/글쓰기는 회원 전용.
- **로그인 유도 패턴**: 라우트 가드 X, 액션 단위 로그인 시트 ○ (토스 스타일).
- **모듈 분리**: 기존 `post` 모듈을 `shared/recruitment/tips`로 가르고 `feed`를 형제 하위 모듈로 추가.
- **L2 호환성 핵심 3종** (지금 안 만들어도 막지는 않는다):
  1. `Post.authorMemberId` nullable 확정
  2. 응답 DTO에 `author` 객체 추상화
  3. `ReportType` enum에 POST/COMMENT 대상 미리 정의

### L2 도입 트리거 (운영 4~6주 후)

다음 중 하나라도 충족되면 L2 검토 시작:

- 비로그인 글쓰기 버튼 클릭 후 이탈률 > 70%
- 비로그인 트래픽 > 30%인데 회원가입 전환은 < 5%
- 사용자 피드백에서 "회원가입 없이 글 쓰고 싶다" 반복 등장

## 실행 순서 (의존성 그래프)

```
A1, A2 (리팩터링 / 스키마 정리)
  ↓
B1~B4 (인증 인프라)
  ↓
C1~C6 ──── D1~D4 ──── E1~E4   (백엔드 도메인, 병렬 가능)
  ↓
F1~F4 (FE 인프라)
  ↓
G1~G7 (FE 화면)
  ↓
H1~H3 (측정)
```

**병렬 가능 지점**: C/D/E는 백엔드 안에서 병렬 진행 가능. FE는 백엔드 API 명세만 합의되면 모킹으로 먼저 시작 가능.
